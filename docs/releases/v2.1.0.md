# 00SWIFT v2.1.0 — Production hardening

This release turns the initial SWIFT/e-CNY prototype into a substantially safer and more maintainable public sandbox. It focuses on authentication integrity, double-entry accounting correctness, reproducible quality gates, and mature project documentation.

## Highlights

- **Credential safety:** application secrets are salted PBKDF2 hashes, bearer tokens are stored only as deterministic digests, and one-time secrets are no longer returned by list APIs.
- **OAuth and signing:** stricter Basic authentication, client-bound revocation, JTI replay protection, scope validation, and exact-body signature verification remove several fail-open paths.
- **Audit privacy:** nested credentials, assertions, tokens, and signatures are recursively redacted; logged bodies are size-bounded.
- **Ledger correctness:** wallet balances and outstanding issuance are derived from balanced ledger entries, including central-bank liability postings for mint and burn operations.
- **Domain validation:** non-finite monetary values, invalid limits, unsafe defaults, and multiple API edge cases are rejected consistently.
- **Reliability:** readiness checks now verify database access, certificate paths are honored, generated private keys are permission-restricted, and startup fails explicitly when required routers cannot load.
- **Regression coverage:** the suite expands from 36 to 50 tests, including OAuth, API security, e-CNY flows, and previously latent regressions.
- **CI/CD:** Python 3.11 and 3.12 tests, a 68% coverage floor, Ruff, Bandit, compile checks, JavaScript parsing, Docker smoke tests, CodeQL, dependency auditing, and release validation are now automated.
- **Repository maturity:** the project now includes an Apache-2.0 license, security policy, contribution guide, code of conduct, threat model, architecture/API/operations documentation, Dependabot, and structured issue/PR templates.

## Verification

The release is published only after the corresponding `main` CI workflow succeeds and CodeQL succeeds for the same commit. The publisher then reruns the quality, security, and coverage gates, creates a source archive, records the resolved dependency set, and uploads SHA-256 checksums.
