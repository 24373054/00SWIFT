# 00SWIFT v4.0.0 — Cross-Border Payment Infrastructure Lab

Version 4.0.0 introduces the first complete institutional operating interface for the 00SWIFT research platform.

## Institutional product experience

The primary interface is now a Vite, React 19 and strict TypeScript application designed for government agencies, banks, CIPS and institutional partners. It uses a restrained graphite, ivory and burgundy visual system, supports Daylight and Operations Dark themes, and provides complete Chinese and English interface resources.

The application is organised by operating workflow rather than protocol brand:

- Command Center
- Payments
- Investigations
- Settlement
- Standards
- Digital Currency
- Risk & Policy
- Developer
- Administration

The previous developer console remains available under `/legacy` during the migration window.

## Operational visualisation suite

The release includes eight connected visualisations that use the same domain state in live and replay modes:

1. Semantic 3D cross-border settlement network with geographic, infrastructure and hybrid views.
2. UETR lifecycle with causal events, SLA state and evidence inspection.
3. RTGS liquidity positions, queue pressure and incoming-payment release dependencies.
4. DNS bilateral obligations, net positions and liquidity-saving calculation.
5. Atomic PvP quote, policy, reserve, lock, commit and rollback sequence.
6. Multi-CBDC jurisdiction policy matrix with amount, purpose and data-residency explanations.
7. ISO 20022 structure explorer with profile findings and direct field navigation.
8. Transaction Copy diff and hash-chain integrity inspection.

The spatial renderer distinguishes message movement, value reservation, queueing, policy blocking and final settlement. It includes complete 2D and Reduced Motion fallbacks and does not rely on decorative particles, neon effects or a generic 3D globe.

## Replay and institutional scenarios

The same visual components support deterministic operational replay. Initial scenarios cover:

- normal cross-border settlement;
- RTGS liquidity shortage and subsequent queue release;
- PvP second-leg failure and atomic rollback.

Each scenario has a virtual clock, state markers, expected final state and system-level causal events.

## Backend-authoritative access control

New institutional UI APIs provide:

- role-aware workspace bootstrap;
- persisted RBAC/ABAC decisions;
- decision IDs and explanations;
- action obligations such as dual approval and operator-reason capture;
- masking and redaction instructions;
- Command Center read models with live database overlays.

Hidden controls are not treated as authorization. Protected operations remain backend-enforced.

## Build, CI and release integrity

The frontend is built in an isolated Node 22 container stage and the verified immutable bundle is served by FastAPI. CI now requires:

- strict frontend typechecking;
- frontend unit tests;
- production bundle generation and artifact verification;
- Python lint, formatting, security and coverage gates;
- SQLite and PostgreSQL migration validation;
- CodeQL analysis;
- container build, readiness, institutional UI, legacy UI and API smoke tests.

Release packages contain the built frontend bundle. Independent post-publication validation rebuilds the frontend and backend from the immutable tag, replaces release assets and verifies their SHA-256 checksums.

## Research and certification boundary

00SWIFT remains an independent research sandbox. This release does not claim official SWIFT, CIPS, People's Bank of China, HKMA, BIS Innovation Hub, mBridge or regulatory certification, authorization or production connectivity. It must not be represented as an official clearing, settlement or central-bank production console.
